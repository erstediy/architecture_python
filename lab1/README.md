# Лабораторная работа 1: Документирование архитектуры в Structurizr DSL

**Вариант 24 — Система управления недвижимостью (аналог Zillow)**

## Роли пользователей

| Роль | Описание |
|------|----------|
| **Покупатель / Арендатор** | Поиск объектов по городу и цене, запись на просмотр. |
| **Агент / Владелец** | Публикация объектов, изменение статуса объекта. |
| **Администратор** | Управление пользователями, объектами и просмотрами. Полный доступ. |

## Внешние системы

| Система | Описание |
|---------|----------|
| **Система уведомлений** | Внешний сервис отправки email/SMS о записи на просмотр. |

## Компоненты системы

| Контейнер | Технология | Описание |
|-----------|-----------|----------|
| **Auth Service** | Python, FastAPI, JWT | Аутентификация, авторизация (JWT), управление пользователями. Роли: ADMIN, AGENT, BUYER. |
| **Property Service** | Python, FastAPI | Управление объектами недвижимости и записями на просмотр. |
| **Notification Service** | Python, aiokafka | Kafka-консьюмер событий. В production отправляет email/SMS через внешний шлюз. |
| **Database** | PostgreSQL | Единая база данных: пользователи, объекты, просмотры. |

## API (вариант 24)

### Пользователи
- `POST /users` — создание пользователя
- `POST /auth/token` — получение JWT токена
- `GET /users/me` — текущий пользователь
- `GET /users?login={login}` — поиск по логину
- `GET /users?name={mask}` — поиск по маске имени

### Объекты недвижимости
- `POST /properties` — создание объекта (AGENT)
- `GET /properties?city={city}` — поиск по городу
- `GET /properties?min_price={min}&max_price={max}` — поиск по цене
- `GET /properties/user/{user_id}` — объекты пользователя
- `PATCH /properties/{id}/status` — изменение статуса (AGENT, ADMIN)

### Просмотры
- `POST /viewings` — запись на просмотр (BUYER)
- `GET /viewings/property/{property_id}` — просмотры объекта

## Диаграммы

- **System Context (C1)** — роли пользователей, система и внешняя система уведомлений
- **Container Diagram (C2)** — два сервиса, база данных и их связи
- **Dynamic: createUser** — создание пользователя администратором
- **Dynamic: addProperty** — публикация объекта агентом
- **Dynamic: scheduleViewing** — запись покупателя на просмотр с отправкой уведомления
